    Welcome to HyperSHIP¶

    HYbrid Power EmulatoR for SHip Propulsion Systems (HyperSHIP)

    hypership is an open-source python package for dynamic simulation and control desing of hybrid shipborad propulsion systems.

    Supported configurations1:

    • Hybrid-Mechanical
    • Hybrid-Electrical

    import numpy as np
import matplotlib.pyplot as plt
import hypership as hs

sim = hs.Simulator(dt=1e-3,t_end=6,type='hy_mech',ctrl_sample_time=1e-3)
current_meas = hs.components.Measurement_3ph(name='line_current')
mg = hs.components.IM(name='M/G',V_ph_n_max=1000)

@hs.utils.input_signal_decorator(signal_type='constant',value=2870)
def ref(t,u): # [rpm]
    return u

@hs.utils.input_signal_decorator(signal_type='step',init_value=0,step_value=8,step_time=3)
def dstrb(t,u): # [Nm]
    return u

bw = 2*np.pi*100; Kp_d, Kp_q, Ki_dq = bw*mg.L_sigma, bw*mg.L_sigma, bw*mg.R_s

i_ds_ctrl = hs.control.PI(sample_time=sim.dt,Kp=Kp_d,Ti=Kp_d/Ki_dq,anti_windup_type='clamping')
i_qs_ctrl = hs.control.PI(sample_time=sim.dt,Kp=Kp_q,Ti=Kp_q/Ki_dq,anti_windup_type='clamping')
im_speed_pi = hs.control.PI(sample_time=sim.ctrl_sample_time,Kp=10,Ti=0.1,anti_windup_type='back-calculation')

with hs.utils.timer():
    sim.run_IM_template_case(case_components={'mg': mg,'current_meas': current_meas},
                             load_torque=dstrb,
                             speed_ref=ref,
                             i_d_ctrl=i_ds_ctrl,
                             i_q_ctrl=i_qs_ctrl,
                             speed_ctrl=im_speed_pi
                             )

hs.scope.do_im_plot(sim_case=sim,start_time_index=np.argmax(sim.t_values >= 0.0),save_path=hs.PATH_OUTPUT_FIGS,timestamp=hs.timestamp,save_fig=True)
plt.show(block=True)

    Getting started¶

    Prerequisites:¶

    • An installation of poetry is needed. 2

    How to use hypership (developement)¶

    1. ssh://git@git-ssh.devops.wartsila.com:7999/~sch091/hypership.git (clone the repository in a local directory)
    2. cd hypership/ (go to the cloned local directory → includes pyproject.toml)

    3. Check python-3.13.1 installation:

    4. IF using pyenv:

      1. pyenv install 3.13.1
      2. pyenv local 3.13.1

    5. ELSE:

      • poetry env use /full/path/to/python.exe

    6. poetry install (creates a new virtual environment with all requirements installed)

    Validate installation:
    poetry run python examples/im_template.py

    Package structure¶

    Module Description
    simulation.py simulation coordinator
    components.py collection of physical components
    control.py collection of controllers
    scope.py results visualization
    utils.py various utility functions
